# PhenoLeaf-TS — Dataset Card

## Summary

| Field | Value |
|-------|-------|
| Images | 17,082 top-down RGB |
| Genotypes | 21 *Arabidopsis thaliana* |
| Plant replicates | 318 (each a temporal sequence) |
| Resolution | ~530 × 525 px |
| Frames / replicate | 53.7 (mean) |
| Leaves / image | 9.7 (mean), range 4–26 |
| Growth stages | 3 (Early / Intermediate / Mature) |
| Instance colours | 31-colour fixed palette |
| License | _to be confirmed (e.g. CC BY-NC 4.0)_ |

## Acquisition

Plants were grown in an environmentally controlled growth chamber and imaged from above at regular
intervals (2 or 4 images/day) with a 4.92 MP RGB camera on an automated gantry, capturing the full
growth trajectory from seedling emergence to a mature rosette. The plant is centred against a soil
background.

## Annotation

Leaf instance masks were manually annotated in CVAT using a fixed palette of 31 perceptually distinct
colours. Leaves are coloured in a deterministic order of appearance (first leaf = colour 1, etc.); a
new leaf takes the next colour in the sequence. **The same physical leaf keeps the same colour across
all frames in which it appears**, so tracking can be evaluated directly against the colour-coded ground
truth without a Hungarian matching step. All annotations passed a two-pass quality-control review.

## Directory structure

```
data/leaf_dataset_colour/
├── 01_raw_dataset/<genotype>/<replicate>/<frame>.png    # RGB input
├── 02_leaf_labels/<genotype>/<replicate>/<frame>.png    # colour-coded instance mask (same size)
└── leaves_unique_colours.xlsx                            # 31-colour palette (leaf id → RGB)
```
Frames within a replicate are temporally ordered by filename.

## Growth stages

Labels are derived automatically from the instance masks by counting unique leaf regions per frame:

| Stage | Leaf count | Share |
|-------|-----------|-------|
| Early | 4–6 | 22.6 % |
| Intermediate | 7–10 | 39.8 % |
| Mature | 11+ | 37.6 % |

## Splits

**70 / 15 / 15** train/val/test, applied within each replicate's temporal sequence. Generated by
`code/prepare_data.py`, which writes `split_standard.json` along with the COCO/YOLO annotations and
growth-stage labels.
